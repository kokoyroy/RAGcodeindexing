## ⚠️ Important Notice

> **This is the `release` branch - DO NOT use directly**

This branch contains the indexer TypeScript source code that gets downloaded from GitHub releases. You should not clone or use this branch directly.

---

## 🎯 Purpose

The `release` branch stores the indexer source code for distribution:

- `src/` - TypeScript source files
- `package.json` - Dependencies (tree-sitter, better-sqlite3, etc.)
- `tsconfig.json` - TypeScript configuration

The installer in `main` branch downloads this code from GitHub release tags, then runs `npm install && npm run build` to compile it.

**Note:** `dist/` is NOT committed to git - it's built by the installer after download.

## 🛠️ For Maintainers

### Creating a New Release

1. **Develop in `develop` branch**
   ```bash
   git checkout develop
   # Make changes, test thoroughly
   ```

2. **Merge to `release` branch**
   ```bash
   git checkout release
   git merge develop
   git add .
   git commit -m "Release v1.0.x"
   ```

3. **Create GitHub release tag**
   ```bash
   git tag v1.0.x
   git push origin release --tags
   ```

4. **Users can now update**
   The installer in `main` branch will detect and download the new release.

## 🔄 How It Works

```
1. Code developed in 'develop' branch
         ↓
2. Merged to 'release' branch (src/ only, no dist/)
         ↓
3. Tagged with version (v1.0.0)
         ↓
4. Installer in 'main' branch downloads from tag
         ↓
5. Installer runs: npm install && npm run build
         ↓
6. dist/ generated locally on user's machine
         ↓
7. User's machine runs the built code
```
